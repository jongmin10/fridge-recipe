# PRD Step 3 — 사용자 프로필 및 레시피 저장

## 목표
사용자 프로필을 생성하고 마음에 드는 레시피를 저장하여 나만의 레시피북을 구성한다.

---

## 기능 요구사항

### 사용자 프로필
- 이메일 + 비밀번호 회원가입 / 로그인
- 프로필 정보:
  - 닉네임
  - 식이 제한 (채식, 글루텐 프리, 유제품 제외 등) — Step 2 레시피 생성 시 프롬프트에 반영
  - 선호 요리 스타일 (한식, 양식, 중식 등)

### 레시피 저장
- Step 2의 레시피 카드에서 "저장하기" 버튼으로 저장
- 저장 시 포함 데이터:
  - 레시피 전체 내용 (이름, 재료, 조리법)
  - 원본 냉장고 이미지 썸네일
  - 저장 날짜

### 저장된 레시피 목록 (레시피북)
- 저장한 레시피를 카드 그리드로 표시
- 검색: 요리 이름으로 검색
- 필터: 조리 시간, 난이도, 요리 스타일
- 레시피 삭제

### 식이 제한 → 레시피 개인화
- 로그인 상태에서 레시피 생성 시 사용자의 식이 제한 정보를 프롬프트에 자동 포함
- 예: "사용자는 채식주의자입니다. 육류를 포함하지 않는 레시피만 추천하세요."

## API 연동

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/auth/signup` | POST | 회원가입 |
| `/api/auth/login` | POST | 로그인 |
| `/api/profile` | GET / PUT | 프로필 조회 및 수정 |
| `/api/recipes/saved` | GET | 저장된 레시피 목록 |
| `/api/recipes/saved` | POST | 레시피 저장 |
| `/api/recipes/saved/:id` | DELETE | 레시피 삭제 |
