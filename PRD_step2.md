# PRD Step 2 — 레시피 추천

## 목표
Step 1에서 추출한 재료 목록을 바탕으로 AI가 만들 수 있는 요리의 레시피를 추천한다.

## 사용 모델
`nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free` (OpenRouter)

---

## 기능 요구사항

### 레시피 생성
- Step 1의 재료 목록을 컨텍스트로 포함한 프롬프트 전송
- 추천 레시피 3개 생성
- 각 레시피 포함 정보:
  - 요리 이름
  - 예상 조리 시간
  - 난이도 (쉬움 / 보통 / 어려움)
  - 필요 재료 (보유 재료 / 추가 필요 재료 구분)
  - 단계별 조리 방법
- 반환 형식 예시:
  ```json
  {
    "recipes": [
      {
        "name": "달걀볶음밥",
        "time": "15분",
        "difficulty": "쉬움",
        "available_ingredients": ["달걀", "당근", "양파"],
        "missing_ingredients": ["밥"],
        "steps": ["팬에 기름을 두른다", "달걀을 볶는다", "..."]
      }
    ]
  }
  ```

### 필터 및 옵션
- 조리 시간 필터: 15분 이하 / 30분 이하 / 제한 없음
- 난이도 필터: 쉬움 / 보통 / 어려움
- 재생성 버튼: 동일 재료로 다른 레시피 추천 재요청

### 레시피 카드
- 요리 이름, 시간, 난이도를 카드 형태로 표시
- 카드 클릭 시 상세 조리 방법 펼침
- 보유/부족 재료를 색상으로 구분 표시

## API 연동

- **엔드포인트**: `POST /api/recipes`
- **요청**:
  ```json
  {
    "ingredients": ["달걀", "우유", "당근"],
    "filters": { "max_time": 30, "difficulty": "쉬움" }
  }
  ```
- **응답**: 레시피 배열 JSON
